object JsonMatcher

Patternmatcher for json values. Use the matchJson method to match json values.

Source
JsonMatcher.scala
Example: 

  1. package test

import org.specs2.mutable._

import no.samordnaopptak.json.J
import no.samordnaopptak.json.JsonMatcher._

class JsonMatcherSpec extends Specification {
  "Json matcher" should {
    "match object with object" in {
      matchJson(
        J.obj(
          "a" -> J.obj(
            "c" -> ___anyString
            "e" -> ___anyArray
          )
        ),
        J.obj(
          "a" -> J.obj(
            "d" -> "astring",
            "e" -> J.arr(2,3,4,5)
          )
        )
      )
    }
  }
}
Note

JsonMatcher provides several custom matchers such as ___anyString, ___anyNumber, RegExp, Custom, etc. All of these matchers have examples.

,

The matcher and json values do not have to be JObject valuess (the result of calling 'J.obj'). matchJson understands a number of different types such as JValue, JsValue (The play framework version of JValue), Map, String, Boolean, etc.

See also

https://github.com/sun-opsys/doppelauge/blob/master/test/JsonMatcherSpec.scala for more examples

  26. def matchJson(matcher: Any, json: Any, throwException: Boolean = true, ignoreArrayOrder: Boolean = false): Boolean

    Matches two json values.

    Matches two json values.

    matcher

    This json value can contain special values such as ___anyString or ___anyBoolean

    The value does not have to be a JValue. 'matchJson' understands a number of different types such as JsValue (The play framework version of JValue), Map, String, etc.

    json

    This is the json value we want to check against the pattern of 'matcher'

    The value does not have to be a JValue. 'matchJson' understands a number of different types such as JsValue (The play framework version of JValue), Map, String, etc.

    throwException

    If true, a JsonMatcherException exception will be thrown if matching fails.

    ignoreArrayOrder

    When matching arrays, the order within the arrays will not be checked.

    The difference between 'ignoreArrayOrder' and the ___ignoreOrder matcher is that 'ignoreArrayOrder' works recursively, while ___ignoreOrder only works for the currently matched array.

    returns

    true if the patterns matched, false if not

    Note

    Using the ___ignoreOrder matcher will have no effect if 'ignoreArrayOrder' is true.
